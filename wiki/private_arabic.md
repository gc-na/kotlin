# الخاصية "private" في كوتلن: التحكم في الوصول

## الملخص
تعتبر الخاصية "private" في كوتلن واحدة من أهم أدوات التحكم في الوصول، حيث تسمح للمطورين بتحديد مستوى الوصول إلى المتغيرات والدوال داخل الفئات.

## الوثائق
تستخدم "private" لتحديد أن المتغير أو الدالة يمكن الوصول إليها فقط داخل الفئة التي تم تعريفها فيها. هذا يساعد في حماية البيانات ويقلل من فرص التلاعب بها من خارج الفئة. عند تعريف عنصر كـ "private"، يصبح غير مرئي للكيانات الأخرى، مما يعزز من مستوى الأمان والتجريد في البرمجة.

### الاستخدام
لإعلان متغير أو دالة كـ "private"، يتم استخدام الكلمة المفتاحية "private" قبل تعريف العنصر. على سبيل المثال:

```kotlin
class MyClass {
    private var myVariable: Int = 0

    private fun myFunction() {
        // بعض العمليات
    }
}
```

في المثال أعلاه، المتغير `myVariable` والدالة `myFunction` لا يمكن الوصول إليهما من خارج `MyClass`.

## الأمثلة
### مثال 1: استخدام الخاصية "private" مع المتغيرات
```kotlin
class User {
    private var password: String = "secret"

    fun showPassword() {
        println(password)
    }
}

fun main() {
    val user = User()
    user.showPassword()  // سيظهر "secret"
    // user.password  // هذا سيسبب خطأ لأن password خاص
}
```

### مثال 2: استخدام الخاصية "private" مع الدوال
```kotlin
class Calculator {
    private fun add(a: Int, b: Int): Int {
        return a + b
    }

    fun calculateSum(a: Int, b: Int): Int {
        return add(a, b)
    }
}

fun main() {
    val calculator = Calculator()
    println(calculator.calculateSum(5, 10))  // سيظهر 15
    // calculator.add(5, 10)  // هذا سيسبب خطأ لأن add خاص
}
```

## الشرح
من الشائع أن يواجه المبتدئون صعوبة في فهم استخدام الخصائص الخاصة. أحد الأخطاء الشائعة هو محاولة الوصول إلى المتغيرات أو الدوال الخاصة من خارج الفئة، مما يؤدي إلى أخطاء وقت التشغيل. بالإضافة إلى ذلك، يجب أن تكون حذرًا عند استخدام الخصائص الخاصة مع الوراثة، حيث لا يمكن الوصول إلى العناصر الخاصة من الفئات الفرعية.

## ملخص من جملة واحدة
تساعد الخاصية "private" في كوتلن على حماية المتغيرات والدوال من الوصول الخارجي، مما يعزز من سلامة الكود وأمانه.
