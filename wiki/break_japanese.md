# Kotlinの「break」文：ループを抜けるための使い方

## 概要
Kotlinの「break」文は、ループ（for、while、do-while）を即座に終了させるための制御フロー文です。この文を使用することで、特定の条件が満たされた際にループを効率的に終了できます。

## ドキュメンテーション
### 目的
「break」文は、ループ内で特定の条件が成立した場合に、そのループを終了させるために使用されます。これにより、不要な繰り返しを避け、プログラムの効率性を向上させることができます。

### 使用法
「break」は、次のように使用します。

```kotlin
for (i in 1..10) {
    if (i == 5) {
        break // iが5のとき、ループを終了
    }
    println(i)
}
```

この例では、iの値が5になった時点でループが終了し、1から4までの数値が出力されます。

### 詳細
- **スコープ**: 「break」は、最も内側のループのみを終了させます。ネストされたループがある場合、外側のループには影響を与えません。
- **条件付き終了**: 複雑な条件に基づいてループを終了させることができ、プログラムの柔軟性を高めます。

## 例
### 基本的な使用例
```kotlin
var sum = 0
for (i in 1..10) {
    if (i > 5) {
        break // iが5を超えたらループを終了
    }
    sum += i
}
println("合計: $sum") // 出力: 合計: 15
```

### ネストしたループでの使用
```kotlin
for (i in 1..3) {
    for (j in 1..3) {
        if (j == 2) {
            break // 内側のループを終了
        }
        println("i: $i, j: $j")
    }
}
```
この場合、出力は以下のようになります。
```
i: 1, j: 1
i: 2, j: 1
i: 3, j: 1
```

## 説明
### よくある落とし穴
- **スコープの誤解**: 「break」は最も内側のループのみを終了します。外側のループにも影響を及ぼすと誤解しないようにしましょう。
- **無限ループ**: 条件を誤って設定すると、ループが終了しない場合があります。特に「while」ループでは、条件が常に真であると無限ループに陥ることがあります。

### 追加の注意点
- Kotlinでは、ループを終了する他に、特定の条件を満たす場合にループをスキップする「continue」文もあります。これらを使い分けることで、より洗練された制御フローを実現できます。

## 一行まとめ
Kotlinの「break」文は、ループを特定の条件に基づいて即座に終了させるための有用な制御フロー文です。
