# الدالة "fun" في كوتلن: كل ما تحتاج معرفته

## ملخص
تُستخدم كلمة "fun" في كوتلن لتعريف الدوال، مما يتيح للمطورين إنشاء وظائف قابلة للتكرار وتنفيذ التعليمات البرمجية بشكل منظم.

## الوثائق
### الغرض
تعتبر الدالة "fun" الأساس لتعريف الوظائف في كوتلن. تسمح لك بإنشاء دوال يمكن استدعاؤها من أماكن متعددة في البرنامج، مما يسهل كتابة الشيفرة وصيانتها.

### الاستخدام
تبدأ أي دالة في كوتلن بكلمة "fun" متبوعة باسم الدالة، ثم قائمة بالمعاملات (إن وجدت) ونوع القيمة المرتجعة. الشكل العام لتعريف دالة هو:

```kotlin
fun اسم_الدالة(معاملات): نوع_القيمة_المرتجعة {
    // التعليمات البرمجية
}
```

### التفاصيل
- **اسم الدالة**: يجب أن يكون فريدًا وغير مشابه لأسماء الدوال الأخرى في نفس النطاق.
- **المعاملات**: يمكن أن تحتوي الدالة على معاملات متعددة، ويجب تحديد نوع كل منها.
- **نوع القيمة المرتجعة**: يمكن أن تكون الدالة غير مرتجعة (Unit) أو ترجع قيمة من نوع معين.

## الأمثلة
### مثال بسيط
```kotlin
fun greet(name: String): String {
    return "مرحبا، $name!"
}
```

### استدعاء الدالة
```kotlin
fun main() {
    val message = greet("علي")
    println(message) // ستظهر: مرحبا، علي!
}
```

### دالة بدون قيمة مرتجعة
```kotlin
fun printHello() {
    println("مرحبا بكم في كوتلن!")
}
```

### استدعاء الدالة
```kotlin
fun main() {
    printHello() // ستظهر: مرحبا بكم في كوتلن!
}
```

## الشرح
- **الأخطاء الشائعة**: تأكد من عدم استخدام أسماء دوال متكررة، وإذا قمت بتعريف دالة بنفس الاسم، ستظهر لديك مشكلة في التداخل.
- **الملاحظات الإضافية**: يمكن أن تحتوي الدوال على دوال داخلية، مما يسمح بتنظيم الشيفرة بشكل أفضل. كما يمكن استخدام "default parameters" لتوفير قيم افتراضية للمعاملات.

## ملخص من سطر واحد
تُستخدم كلمة "fun" في كوتلن لتعريف الدوال، مما يسهل تنظيم وتنفيذ التعليمات البرمجية بشكل فعال.
