# استخدام الأمر "break" في لغة كوتلن: الدليل الشامل

## ملخص
الأمر "break" في كوتلن يُستخدم لإنهاء حلقة تكرارية (Loop) بشكل فوري، مما يُساعد في التحكم بتدفق البرنامج وتحسين الأداء.

## الوثائق
يُستخدم الأمر "break" في كوتلن لإيقاف تنفيذ حلقة تكرارية مثل `for`, `while`, أو `do..while`. عند استدعاء الأمر "break"، يتم إنهاء الحلقة الحالية والانتقال مباشرة إلى السطر التالي بعد الحلقة.

### الغرض
الهدف من استخدام "break" هو التحكم في سير البرنامج، خاصةً عند الحاجة لإنهاء الحلقة في حالة معينة، مثل العثور على عنصر معين أو عند استيفاء شرط معين.

### الاستخدام
يمكن استخدام الأمر "break" في أي حلقة تكرارية، سواء كانت حلقة `for` أو `while`. يتم استخدامه بالشكل التالي:
```kotlin
break
```

### تفاصيل
- يُعتبر "break" جزءاً من التحكم في تدفق البرنامج.
- يمكن استخدامه داخل الحلقات المتداخلة، وعندها سيؤثر فقط على الحلقة الأقرب.

## الأمثلة
### مثال 1: استخدام "break" في حلقة `for`
```kotlin
for (i in 1..10) {
    if (i == 5) {
        break
    }
    println(i)
}
// الناتج: 1 2 3 4
```

### مثال 2: استخدام "break" في حلقة `while`
```kotlin
var count = 0
while (count < 10) {
    if (count == 3) {
        break
    }
    println(count)
    count++
}
// الناتج: 0 1 2
```

## الشرح
### العثرات الشائعة
- **عدم استخدام "break" بشكل صحيح**: يمكن أن يؤدي استخدام "break" بغير تفكير إلى تخطي أجزاء مهمة من البرنامج.
- **استخدام "break" في الحلقات المتداخلة**: يجب الانتباه إلى أن "break" سيؤثر فقط على الحلقة الأقرب، لذا يجب الحذر عند استخدامه ضمن حلقات متعددة.

### ملاحظات إضافية
- "break" لا يؤثر على الشروط أو التعليمات البرمجية خارج الحلقة.
- يُنصح باستخدام "break" عند الحاجة الفعلية لإنهاء الحلقة، وليس بشكل عشوائي.

## ملخص من جملة واحدة
الأمر "break" في كوتلن يُستخدم لإيقاف تنفيذ الحلقة الحالية والتحكم في تدفق البرنامج بشكل فعال.
