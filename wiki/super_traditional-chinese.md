# Kotlin中的「super」關鍵字：用法與範例

## 概述
在Kotlin中，「super」關鍵字用於呼叫父類別的方法或屬性，特別是在子類別中重寫這些方法或屬性時。這使得開發者能夠更好地利用繼承機制，並在子類別中執行父類別的功能。

## 文件說明
### 目的
「super」關鍵字的主要目的是允許子類別訪問其父類別中的方法和屬性，特別是在這些方法或屬性被子類別重寫的情況下。

### 使用方式
在Kotlin中，當你想要在子類別中呼叫父類別的某個方法時，可以使用「super」關鍵字。這通常用於重寫方法時，以確保父類別的邏輯也能被執行。

### 詳細說明
- **語法**: 在函數內部使用`super`關鍵字，後接父類別的方法名稱。例如：`super.methodName()`
- **注意事項**: 使用「super」時，必須確保所呼叫的方法在父類別中存在，否則將會產生編譯錯誤。此外，當一個方法被多重繼承的類別重寫時，需特別注意正確的父類別調用。

## 範例
以下是「super」關鍵字的基本用法示範：

```kotlin
open class Parent {
    open fun display() {
        println("父類別的方法")
    }
}

class Child : Parent() {
    override fun display() {
        super.display()  // 呼叫父類別的方法
        println("子類別的方法")
    }
}

fun main() {
    val child = Child()
    child.display()
}
```
### 輸出：
```
父類別的方法
子類別的方法
```

## 解釋
- **常見問題**: 使用「super」時，開發者常常會忘記在重寫的方法內呼叫父類別的方法，這可能導致父類別的邏輯無法執行。
- **注意事項**: 在Kotlin中，若一個類別從多個父類別繼承而來，使用「super」時需要明確指定要調用的父類別，如`super<Parent>.methodName()`。

## 總結
「super」關鍵字在Kotlin中是用來訪問父類別的方法或屬性，特別是在子類別重寫這些方法時，確保父類別的邏輯得以執行。
