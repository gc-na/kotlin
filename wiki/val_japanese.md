# Kotlinにおける「val」の使い方と特徴

## 概要
Kotlinにおける「val」は、変更不可の変数を定義するためのキーワードです。これにより、プログラム内で不変のデータを扱うことができ、コードの安全性と可読性を向上させます。

## ドキュメンテーション
「val」は、Kotlinで変数を宣言する際に使用されます。宣言された変数には、一度値が割り当てられると、その後は値を変更することができません。これは、関数内やクラス内、あるいはトップレベルで使用できます。

### 目的
「val」を使用することで、意図しない変更から変数を保護し、プログラムのロジックを明確にすることができます。

### 使用法
以下のように「val」を使って変数を宣言できます。

```kotlin
val constantValue: Int = 10
```

この例では、`constantValue`という名称の整数型の変数が定義され、値は10に設定されています。この変数に新しい値を代入しようとすると、コンパイルエラーが発生します。

### 詳細
- `val`で宣言された変数は、初期化時にのみ値を設定できます。
- 型推論が働くため、型を明示的に指定する必要はありません。例えば、以下のように書くことも可能です。

```kotlin
val name = "Kotlin"
```

- `val`はスレッドセーフであり、並行処理の際にも安全に使用できます。

## 例
以下に「val」の基本的な使用例を示します。

### 例1: 基本的な使用
```kotlin
val pi: Double = 3.14
println(pi) // 出力: 3.14
```

### 例2: 型推論を使用
```kotlin
val message = "Hello, Kotlin!"
println(message) // 出力: Hello, Kotlin!
```

### 例3: オブジェクトの不変性
```kotlin
data class User(val name: String)

val user = User("Alice")
// user.name = "Bob" // エラー: `val`プロパティは変更不可
println(user.name) // 出力: Alice
```

## 説明
「val」を使用する際の一般的な落とし穴や注意点は以下の通りです。

- **再代入の禁止**: 一度値を設定した`val`変数には、後から新しい値を代入することができません。これを誤って行おうとするとコンパイルエラーが発生します。
- **不変オブジェクト**: `val`はオブジェクトの不変性を保証しません。オブジェクトのプロパティが`var`であれば、そのプロパティの値は変更可能です。
- **スコープに注意**: スコープ内で`val`変数が有効であることを理解しておくことが重要です。ブロック内で宣言された`val`は、そのブロックの外ではアクセスできません。

## 一文要約
Kotlinの「val」は、変更不可の変数を宣言するためのキーワードであり、安全で明瞭なプログラミングをサポートします。
