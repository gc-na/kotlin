# Kotlin 函數 (fun) 的深入探討

## 總結
在 Kotlin 中，`fun` 是用來定義函數的關鍵字。函數是編程中的基本構建塊，允許開發者封裝可重用的代碼。

## 文檔
在 Kotlin 語言中，`fun` 關鍵字用來聲明一個函數。函數是一組可以重複執行的代碼，通常接受輸入參數並返回一個結果。函數的主要目的在於提高代碼的可重用性和可讀性。

### 語法
函數的基本語法如下：
```kotlin
fun 函數名稱(參數名稱: 參數類型): 返回類型 {
    // 函數主體
}
```

- **函數名稱**：標識符，用於調用函數。
- **參數**：可以有零個或多個，格式為`名稱: 類型`。
- **返回類型**：默認為`Unit`，如果有返回值則需顯式聲明。

## 示例
以下是一些基本的函數示例：

### 無參數和返回值的函數
```kotlin
fun sayHello() {
    println("你好，世界！")
}
```

### 帶參數的函數
```kotlin
fun add(a: Int, b: Int): Int {
    return a + b
}
```

### 帶默認參數的函數
```kotlin
fun greet(name: String = "朋友") {
    println("你好，$name！")
}
```

### 使用函數
調用函數的示例：
```kotlin
sayHello() // 輸出：你好，世界！
val sum = add(3, 5) // sum 為 8
greet() // 輸出：你好，朋友！
greet("小明") // 輸出：你好，小明！
```

## 解釋
在使用 `fun` 定義函數時，開發者需要注意以下幾點：

- **類型推斷**：Kotlin 支援類型推斷，當函數返回值符合特定類型時，可以省略返回類型的宣告。
- **高階函數**：Kotlin 支持高階函數，即函數可以接受其他函數作為參數或返回函數。
- **可變參數**：可以使用 `vararg` 關鍵字來定義可變參數的函數。
  
常見的陷阱包括：
- 忘記返回值的類型宣告，可能導致編譯錯誤。
- 在函數內部訪問外部變量時，可能會遇到作用域的問題。

## 一句總結
Kotlin 的 `fun` 關鍵字用於定義函數，這使得代碼更具可讀性和可重用性。
