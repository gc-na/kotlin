# الباني (Constructor) في كوتلن: دليل شامل

## ملخص
الباني في كوتلن هو طريقة خاصة تُستخدم لإنشاء كائنات جديدة من الفئات (Classes). يتيح لك تهيئة الكائنات وتحديد القيم الابتدائية للخصائص.

## الوثائق
### الغرض
يهدف الباني إلى تهيئة كائنات الفئة عند إنشائها. يمكن أن يحتوي على معلمات تُستخدم لضبط الخصائص الأولية للكائن.

### الاستخدام
في كوتلن، يمكن تعريف الباني باستخدام الكلمة المفتاحية `constructor`، لكن يمكن أيضًا تعريفه بشكل افتراضي بدون الحاجة لاستخدام الكلمة. يوجد نوعان من البانيات: 
1. **الباني الأساسي (Primary Constructor)**: يُعرَّف مباشرة في رأس الفئة.
2. **الباني الثانوي (Secondary Constructor)**: يُستخدم لتعريف باني إضافي داخل جسم الفئة.

### التفاصيل
- **الباني الأساسي**:
  ```kotlin
  class Person(val name: String, var age: Int)
  ```
  في هذا المثال، `name` هو خاصية ثابتة (val) و`age` خاصية قابلة للتغيير (var).

- **الباني الثانوي**:
  ```kotlin
  class Person(val name: String) {
      var age: Int = 0

      constructor(name: String, age: Int) : this(name) {
          this.age = age
      }
  }
  ```
  هنا، يتم تعريف باني ثانوي يأخذ معلمتين.

## الأمثلة
### مثال على الباني الأساسي
```kotlin
class Car(val model: String, var year: Int)

fun main() {
    val car = Car("Toyota", 2020)
    println("Model: ${car.model}, Year: ${car.year}")
}
```

### مثال على الباني الثانوي
```kotlin
class Book(val title: String) {
    var author: String = ""

    constructor(title: String, author: String) : this(title) {
        this.author = author
    }
}

fun main() {
    val book = Book("Kotlin for Beginners", "John Doe")
    println("Title: ${book.title}, Author: ${book.author}")
}
```

## الشرح
### المزالق الشائعة
- عدم استخدام `this` عند الإشارة إلى الخصائص داخل الباني الثانوي يمكن أن يؤدي إلى أخطاء.
- في حالة عدم تعريف أي باني، سيتم إنشاء باني افتراضي فارغ. إذا كان لديك خصائص غير قابلة للإفتراض، يجب عليك تعريف باني أساسي أو ثانوي لتجنب الأخطاء.

### ملاحظات إضافية
- يمكنك استخدام `init` لتهيئة الخصائص داخل الباني الأساسي.
- يُفضل استخدام الباني الأساسي عند الحاجة إلى تهيئة الخصائص بشكل مباشر.

## ملخص جملة واحدة
الباني في كوتلن هو وسيلة لتخصيص إنشاء الكائنات من الفئات وتحديد القيم الابتدائية للخصائص.
