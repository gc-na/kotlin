# الكلمة المفتاحية "init" في لغة كوتلن: كل ما تحتاج معرفته

## ملخص
كلمة "init" في كوتلن تُستخدم لتعريف المُنشئ (Constructor) في الصفوف (Classes). يسمح المُنشئ بتهيئة كائنات جديدة من الصف وتعيين القيم الأولية للخصائص.

## توثيق
تعتبر "init" جزءًا أساسيًا من تعريف الصفوف في كوتلن، حيث يتم استدعاؤها تلقائيًا عند إنشاء كائن من الصف. تُستخدم "init" لتطبيق منطق تهيئة إضافي يتجاوز مجرد تعيين القيم للخصائص. يمكن أن تحتوي على تعليمات برمجية تُنفذ عند إنشاء الكائن.

### الغرض
الغرض الأساسي من "init" هو تهيئة الحالة الأولية للكائن، مما يضمن أن جميع الخصائص مضبوطة بشكل صحيح قبل استخدام الكائن.

### الاستخدام
لإنشاء مُنشئ باستخدام "init"، يمكن استخدام الكود التالي:
```kotlin
class Person(val name: String) {
    init {
        println("تم إنشاء شخص باسم $name")
    }
}
```
عند إنشاء كائن جديد من الصف `Person`، سيتم استدعاء الكود داخل "init".

## أمثلة
### مثال 1: مُنشئ بسيط
```kotlin
class Car(val model: String) {
    init {
        println("سيارة جديدة من طراز $model تم إنشاؤها.")
    }
}

fun main() {
    val myCar = Car("تويوتا")
}
```

### مثال 2: تهيئة الخصائص
```kotlin
class Rectangle(val width: Double, val height: Double) {
    val area: Double

    init {
        area = width * height
        println("مساحة المستطيل: $area")
    }
}

fun main() {
    val myRectangle = Rectangle(5.0, 3.0)
}
```

## توضيح
### الأخطاء الشائعة
- **نسيان كتابة "init"**: إذا نسيت تعريف "init" في الصف، قد تفوت تنفيذ أي منطق تهيئة مهم.
- **استخدام قيم غير صالحة**: التأكد من صحة القيم المستخدمة في "init" لتفادي الأخطاء أثناء تشغيل البرنامج.

### ملاحظات إضافية
- يمكن أن تحتوي "init" على أكثر من كتلة "init" في الصف، ولكن يجب أن تكون مرتبة بشكل صحيح. يُستحسن تخزين منطق التهيئة في كتلة واحدة لتسهيل الصيانة.
- يُمكن استخدام "init" في الصفوف المجردة (abstract classes) مما يتيح لك توسيع المنطق في الصفوف الفرعية.

## ملخص من جملة واحدة
كلمة "init" في كوتلن تستخدم لتعريف المُنشئ الذي يقوم بتهيئة الكائنات عند إنشائها.
