# Kotlin 인터페이스: 객체 지향 프로그래밍의 핵심 요소

## 개요
Kotlin에서 인터페이스는 클래스가 구현해야 하는 메소드와 프로퍼티의 집합을 정의하는 중요한 구조입니다. 인터페이스를 활용하면 다형성을 제공하고, 객체 간의 상호작용을 더욱 유연하게 만들어줍니다.

## 문서화
Kotlin의 인터페이스는 다음과 같은 특징을 가지고 있습니다:

- **정의**: 인터페이스는 메소드의 시그니처와 프로퍼티를 정의하지만, 실제 구현을 포함하지 않습니다.
- **구현**: 클래스는 하나 이상의 인터페이스를 구현할 수 있으며, 이로 인해 다중 상속의 장점을 얻을 수 있습니다.
- **기본 구현**: Kotlin에서는 인터페이스 내에서 기본 메소드 구현을 제공할 수 있어, 구현하는 클래스가 선택적으로 이 메소드를 오버라이드 할 수 있습니다.

### 사용법
인터페이스는 `interface` 키워드를 사용하여 정의됩니다. 다음은 인터페이스를 정의하고 구현하는 기본적인 예시입니다.

## 예제
```kotlin
// 인터페이스 정의
interface Animal {
    fun sound(): String
}

// 인터페이스 구현
class Dog : Animal {
    override fun sound(): String {
        return "Woof"
    }
}

class Cat : Animal {
    override fun sound(): String {
        return "Meow"
    }
}

// 사용 예시
fun main() {
    val dog: Animal = Dog()
    val cat: Animal = Cat()

    println(dog.sound()) // 출력: Woof
    println(cat.sound()) // 출력: Meow
}
```

## 설명
Kotlin의 인터페이스를 사용할 때 주의해야 할 점은 다음과 같습니다:

1. **다중 상속의 제한**: Kotlin에서는 클래스를 다중 상속할 수 없지만, 여러 인터페이스를 구현할 수 있습니다. 이를 통해 코드 재사용성을 높일 수 있습니다.
2. **기본 구현의 활용**: 인터페이스에서 기본 메소드를 구현하면, 이를 구현하는 클래스에서 해당 메소드를 오버라이드 할 필요가 없어질 수 있습니다. 그러나 기본 구현을 사용할 경우, 명확한 의도를 유지하는 것이 중요합니다.
3. **프로퍼티와 접근자**: 인터페이스 내에서 프로퍼티를 정의할 수 있으며, 이를 통해 객체의 상태를 캡슐화할 수 있습니다. 프로퍼티는 `get`과 `set` 접근자를 통해 구현할 수 있습니다.

## 한 줄 요약
Kotlin의 인터페이스는 클래스가 구현해야 하는 메소드와 프로퍼티를 정의하며, 다형성과 코드 재사용을 촉진하는 객체 지향 프로그래밍의 중요한 요소입니다.
