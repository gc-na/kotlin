# Kotlin에서의 try 문: 예외 처리를 위한 핵심 기능

## 개요
Kotlin의 `try` 문은 프로그램에서 발생할 수 있는 예외를 처리하는 데 사용되는 구조체입니다. 이를 통해 개발자는 오류 발생 시 프로그램의 비정상 종료를 방지하고, 안전하게 에러를 관리할 수 있습니다.

## 문서화
`try` 문은 Kotlin에서 예외를 처리하기 위한 기본적인 방법입니다. 일반적으로 `try` 블록 내에서 발생할 수 있는 예외를 감지하고, 이를 처리하기 위한 `catch` 블록을 함께 사용합니다. 또한, `finally` 블록을 통해 예외 발생 여부와 관계없이 항상 실행되어야 하는 코드를 정의할 수 있습니다.

### 사용법
`try` 문은 다음과 같이 사용됩니다:

```kotlin
try {
    // 위험한 코드 실행
} catch (e: ExceptionType) {
    // 예외 처리 코드
} finally {
    // 항상 실행되는 코드
}
```

- **try**: 예외가 발생할 가능성이 있는 코드를 포함합니다.
- **catch**: 예외가 발생했을 때 실행되는 코드를 포함합니다. 여러 개의 `catch` 블록을 사용하여 다양한 예외를 처리할 수 있습니다.
- **finally**: 예외 발생 여부와 관계없이 항상 실행되어야 하는 코드를 포함합니다. (선택 사항)

## 예제
다음은 `try` 문을 사용하는 기본적인 예제입니다:

```kotlin
fun main() {
    val number: Int

    try {
        number = "Not a number".toInt()
    } catch (e: NumberFormatException) {
        println("예외 발생: ${e.message}")
    } finally {
        println("이 코드는 항상 실행됩니다.")
    }
}
```

위의 예제에서, 문자열을 정수로 변환하는 과정에서 `NumberFormatException`이 발생하고, 이를 `catch` 블록에서 처리합니다. `finally` 블록의 내용은 예외 발생 여부와 관계없이 항상 실행됩니다.

## 설명
Kotlin에서 `try` 문을 사용할 때 주의해야 할 몇 가지 점이 있습니다:

1. **예외의 스코프**: `try` 블록 내에서 발생한 예외는 해당 블록 내에서만 처리됩니다. 외부에서 발생한 예외는 처리되지 않습니다.
2. **여러 개의 catch 블록**: 여러 종류의 예외를 처리하기 위해 여러 개의 `catch` 블록을 사용할 수 있습니다. 각 블록은 상속 관계에 따라 특정 예외부터 일반 예외까지 처리할 수 있습니다.
3. **finally 블록의 필요성**: `finally` 블록은 선택 사항입니다. 필요하지 않으면 생략할 수 있지만, 자원 해제를 위해 사용하는 것이 좋습니다.
4. **에러와 예외**: Kotlin에서 `Error`와 `Exception`은 다릅니다. `Error`는 프로그램의 심각한 문제를 나타내며, 일반적으로 처리할 수 없습니다.

## 한 줄 요약
Kotlin의 `try` 문은 예외를 안전하게 처리하여 프로그램의 비정상 종료를 방지하는 강력한 도구입니다.
