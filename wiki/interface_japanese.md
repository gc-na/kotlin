# Kotlinのインターフェース: 基礎から応用まで

## 概要
Kotlinのインターフェースは、クラス間での共通の機能を定義し、実装を強制するための重要な構造です。これにより、コードの再利用性や拡張性が向上します。

## ドキュメント
Kotlinでは、インターフェースを使用して、メソッドのシグネチャを定義できます。インターフェースは、クラスが特定のメソッドを実装することを要求しますが、実装自体は提供しません。インターフェースは、複数のクラスで共有される機能を定義するために使用されることが多いです。

### 使用法
インターフェースは、`interface`キーワードを使用して定義します。以下はその基本的な構文です。

```kotlin
interface インターフェース名 {
    fun メソッド名(): 戻り値の型
}
```

クラスがインターフェースを実装するには、`:`を使用してインターフェースを指定し、必要なメソッドをオーバーライドします。

```kotlin
class クラス名 : インターフェース名 {
    override fun メソッド名() {
        // 実装
    }
}
```

## 例
以下は、Kotlinのインターフェースの基本的な使用例です。

```kotlin
interface 優雅 {
    fun 表現する()
}

class 猫 : 優雅 {
    override fun 表現する() {
        println("ニャー")
    }
}

class 犬 : 優雅 {
    override fun 表現する() {
        println("ワン")
    }
}

fun main() {
    val 猫のインスタンス: 優雅 = 猫()
    val 犬のインスタンス: 優雅 = 犬()
    
    猫のインスタンス.表現する() // ニャー
    犬のインスタンス.表現する() // ワン
}
```

## 説明
インターフェースを使用する際の一般的な落とし穴として、以下の点に注意が必要です。

- **複数のインターフェースの実装**: Kotlinは多重継承をサポートしていないため、複数のインターフェースを実装する場合は、すべてのメソッドを必ず実装する必要があります。
- **デフォルトメソッド**: Kotlinでは、インターフェース内でデフォルト実装を提供することができますが、これを利用する際は注意が必要です。デフォルトメソッドを同じ名前で持つ異なるインターフェースを実装すると、どの実装を使用するかを明示的に指定する必要があります。
- **プロパティの定義**: インターフェースでは、プロパティを定義することも可能ですが、実装するクラスではゲッターとセッターを実装する必要があります。
