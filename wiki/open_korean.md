# Kotlin의 "open" 키워드: 상속 가능 클래스를 정의하는 방법

## 개요
Kotlin 프로그래밍 언어에서 "open" 키워드는 클래스나 메서드가 상속 가능하게 설정할 수 있도록 하는 키워드입니다. 기본적으로 Kotlin의 클래스는 final로 설정되어 상속할 수 없지만, "open" 키워드를 사용하면 이를 허용할 수 있습니다.

## 문서화

### 목적
"open" 키워드는 클래스나 메서드가 다른 클래스에 의해 상속될 수 있도록 허용하는 데 사용됩니다. 이를 통해 객체 지향 프로그래밍의 상속 개념을 활용하여 코드의 재사용성을 높일 수 있습니다.

### 사용법
- **클래스 정의에서의 사용**: 클래스를 정의할 때 "open" 키워드를 추가하면 해당 클래스는 다른 클래스에서 상속될 수 있습니다.
- **메서드 정의에서의 사용**: 메서드를 정의할 때 "open" 키워드를 추가하면 해당 메서드는 서브클래스에서 오버라이드(재정의)할 수 있습니다.

### 예제
1. **클래스 상속 예제**:
   ```kotlin
   open class Animal {
       open fun sound() {
           println("Animal sound")
       }
   }

   class Dog : Animal() {
       override fun sound() {
           println("Bark")
       }
   }
   ```

2. **메서드 오버라이드 예제**:
   ```kotlin
   open class Vehicle {
       open fun start() {
           println("Vehicle starting")
       }
   }

   class Car : Vehicle() {
       override fun start() {
           println("Car starting")
       }
   }
   ```

## 설명
- **기본 클래스는 final**: Kotlin에서는 클래스가 명시적으로 "open"으로 설정되지 않는 한 기본적으로 final입니다. 따라서 상속을 원할 경우 반드시 "open" 키워드를 사용해야 합니다.
- **상속의 필요성**: 상속을 통해 코드의 중복을 줄이고, 공통 기능을 부모 클래스에서 정의할 수 있습니다. 이를 통해 유지 관리가 용이해집니다.

### 흔히 발생하는 오류 및 주의 사항
- "open" 키워드를 잊고 클래스나 메서드를 정의하는 경우, 해당 클래스나 메서드는 상속할 수 없게 되어 원하는 기능을 구현할 수 없습니다.
- 다중 상속은 지원하지 않지만, 인터페이스를 사용하여 유사한 기능을 구현할 수 있습니다.

## 한 줄 요약
Kotlin의 "open" 키워드는 클래스와 메서드를 상속 가능하게 설정하여 객체 지향 프로그래밍의 재사용성을 높이는 데 사용되는 키워드입니다.
