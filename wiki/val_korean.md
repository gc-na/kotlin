# Kotlin의 val: 변수 선언의 필수 요소

## 개요
Kotlin의 `val` 키워드는 불변 변수를 선언하는 데 사용됩니다. 이는 변수가 초기화된 후 값을 변경할 수 없음을 의미하며, 코드의 안정성과 가독성을 높이는 데 기여합니다.

## 문서화
Kotlin에서의 `val`은 변수를 선언할 때 사용되는 키워드로, 변수가 한 번 초기화된 후에는 값을 변경할 수 없도록 합니다. 이는 코드에서 불변성을 유지하고, 상태를 변경할 수 없는 객체를 만들 수 있게 해줍니다. `val`로 선언된 변수를 사용하는 것은 불변 객체의 이점을 취할 수 있게 해주며, 멀티스레드 환경에서도 안전한 코드 작성을 가능하게 합니다.

### 사용법
`val` 키워드는 다음과 같이 사용됩니다:
```kotlin
val 변수명: 데이터타입 = 초기값
```
여기서 `변수명`은 변수를 식별하는 이름이며, `데이터타입`은 변수에 저장될 값의 유형을 지정합니다. 초기값은 변수의 초기값을 설정합니다.

## 예제
```kotlin
// 정수형 변수
val number: Int = 10

// 문자열 변수
val name: String = "Kotlin"

// 리스트 변수
val numbersList: List<Int> = listOf(1, 2, 3, 4, 5)
```
위 예제에서는 `val`을 사용하여 각각 정수형, 문자열형, 리스트형 변수를 선언했습니다. 이 변수들은 초기화 후 값이 변경될 수 없습니다.

## 설명
`val`을 사용할 때 주의해야 할 점은 다음과 같습니다:

- **불변성과 가변성**: `val`로 선언된 변수는 불변이지만, 그 변수에 참조된 객체가 가변인 경우 객체의 속성은 변경될 수 있습니다. 예를 들어, 리스트를 `val`로 선언하면 리스트 자체는 변경할 수 없지만, 리스트의 요소는 변경할 수 있습니다.
  
  ```kotlin
  val mutableList: MutableList<Int> = mutableListOf(1, 2, 3)
  mutableList.add(4) // 가능
  ```

- **초기화**: `val` 변수는 선언 시 반드시 초기화되어야 하며, 나중에 값을 다시 할당할 수 없습니다. 초기화하지 않으면 컴파일 오류가 발생합니다.

- **타입 추론**: Kotlin은 타입 추론 기능을 제공하므로, 변수의 타입을 명시적으로 지정하지 않고도 초기값으로부터 타입을 유추할 수 있습니다.

  ```kotlin
  val inferredType = "Kotlin" // String으로 추론됨
  ```

## 한 줄 요약
Kotlin의 `val` 키워드는 불변 변수를 선언하여 코드의 안정성을 높이는 데 중요한 역할을 합니다.
