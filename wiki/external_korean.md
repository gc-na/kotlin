# Kotlin의 "external" 키워드: 외부 함수 및 속성 사용법

## 개요
Kotlin의 "external" 키워드는 외부 코드(예: C/C++ 라이브러리)와의 상호작용을 가능하게 하는 기능입니다. 이를 통해 Kotlin 코드에서 네이티브 라이브러리의 함수와 속성을 호출할 수 있습니다.

## 문서화
### 목적
Kotlin의 "external" 키워드는 코드베이스 내에서 네이티브 라이브러리와의 연동을 쉽게 해주는 역할을 합니다. 이 키워드를 사용하면 Kotlin 코드에서 직접적으로 C/C++로 구현된 함수를 사용할 수 있습니다.

### 사용법
"external" 키워드는 네이티브 함수를 정의할 때 사용합니다. 외부 함수는 구현 내용이 없으며, 호출될 때 네이티브 라이브러리에서 실행됩니다. 기본적인 구문은 다음과 같습니다:

```kotlin
external fun 함수이름(매개변수: 타입): 반환타입
```

### 세부사항
- "external" 함수는 반드시 Kotlin/Native로 컴파일되어야 하며, JNI(Java Native Interface)와는 다릅니다.
- "external" 속성도 지원되며, 이는 네이티브 메모리에서 직접 값을 읽고 쓸 수 있게 해줍니다.
- 네이티브 라이브러리는 링크 단계에서 제공되어야 하며, Kotlin 코드에서 참조할 수 있도록 설정해야 합니다.

## 예제
아래는 "external" 키워드를 사용하여 C 라이브러리의 함수를 호출하는 간단한 예제입니다.

```kotlin
// 외부 함수 선언
external fun 외부함수이름(숫자: Int): Int

fun main() {
    // 외부 함수를 호출하여 결과를 출력
    val 결과 = 외부함수이름(5)
    println("결과: $결과")
}
```

## 설명
- **일반적인 함정**: "external" 키워드를 사용할 때, 해당 함수가 실제로 존재하는지, 라이브러리가 올바르게 링크되었는지 확인해야 합니다. 
- **가독성**: "external" 함수는 구현이 없기 때문에, 함수의 기능을 이해하기 위해서는 외부 문서나 주석을 통해 설명을 제공하는 것이 좋습니다.
- **플랫폼 의존성**: "external" 키워드는 특정 플랫폼에 의존적일 수 있으며, 이식성이 떨어질 수 있습니다.

## 한 줄 요약
Kotlin의 "external" 키워드는 외부 C/C++ 라이브러리의 함수와 속성을 호출할 수 있게 해주는 기능입니다.
