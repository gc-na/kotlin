# استخدام الكلمة المحجوزة "const" في كوتلن

## الملخص
تعتبر الكلمة المحجوزة "const" في كوتلن وسيلة لتعريف الثوابت في مستوى أعلى من التهيئة، مما يسمح للمبرمجين بتعريف قيم ثابتة يمكن استخدامها في جميع أنحاء البرنامج دون تغيير.

## الوثائق
تستخدم الكلمة المحجوزة "const" في كوتلن لتعريف متغيرات ثابتة من نوع بدائي. يمكن استخدام "const" فقط مع المتغيرات التي تُعرّف في مستوى أعلى (top-level) أو داخل الكائنات (objects) المُعرفة كـ "companion objects". يجب أن تكون القيم المعينة لـ "const" عبارة عن تعبيرات ثابتة يتم تقييمها في وقت الترجمة (compile-time) وليس وقت التنفيذ.

### الاستخدام
لإنشاء ثوابت باستخدام "const"، يجب عليك استخدام الكلمة "const" قبل تعريف المتغير. إليك كيفية استخدامها:

```kotlin
const val MY_CONSTANT = "Hello, Kotlin!"
```

تُستخدم "const" لتسريع الوصول إلى الثوابت في البرنامج، حيث تُعتبر هذه الثوابت جزءًا من الكود المترجم ولا تتطلب تخصيصًا إضافيًا في الذاكرة.

## الأمثلة
### مثال 1: تعريف ثابت نصي
```kotlin
const val GREETING = "مرحبًا بك في كوتلن"
fun main() {
    println(GREETING)
}
```

### مثال 2: تعريف ثابت عددي
```kotlin
const val MAX_USERS = 100
fun main() {
    println("عدد المستخدمين الأقصى هو: $MAX_USERS")
}
```

### مثال 3: استخدام الثوابت في دالة
```kotlin
const val PI = 3.14
fun calculateCircleArea(radius: Double): Double {
    return PI * radius * radius
}
fun main() {
    println("مساحة الدائرة: ${calculateCircleArea(5.0)}")
}
```

## الشرح
### الفخاخ الشائعة
- **عدم قابلية التغيير**: يجب أن تضع في اعتبارك أن الثوابت المعرفة بواسطة "const" لا يمكن تغيير قيمتها بعد تعريفها، لذا تأكد من تعيينها للقيم المناسبة منذ البداية.
- **استخدام التعبيرات الثابتة**: يمكنك فقط استخدام تعبيرات ثابتة (مثل الأعداد والنصوص البسيطة) مع "const"، لذا حاول تجنب استخدام كائنات أو تعبيرات ديناميكية.
- **نطاق الأسماء**: الثوابت المعرفة باستخدام "const" لها نطاق أسماء ثابت، لذا تأكد من تسميتها بشكل فريد لتجنب التصادم مع أسماء أخرى في نفس النطاق.

## ملخص جملة واحدة
تسمح الكلمة المحجوزة "const" في كوتلن بتعريف ثوابت ذات قيم ثابتة يمكن استخدامها على مستوى البرنامج بشكل فعال وسريع.
