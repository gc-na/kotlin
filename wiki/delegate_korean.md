# 위임(Delegate) - 코틀린에서의 활용과 이해

## 개요
코틀린의 위임(delegate) 기능은 객체의 속성이나 메서드 호출을 다른 객체에 위임하는 강력한 메커니즘입니다. 이는 코드의 재사용성과 유지보수성을 높이는 데 기여합니다.

## 문서화
위임은 코틀린에서 매우 유용한 기능으로, 특정 속성의 값을 다른 객체에 위임할 수 있도록 지원합니다. 이를 통해 객체 간의 관계를 명확하게 정의하고, 코드의 중복을 줄일 수 있습니다. 코틀린에서는 두 가지 주요 형태의 위임을 제공합니다:

1. **속성 위임**: `by` 키워드를 사용하여 속성의 값을 다른 객체에 위임합니다.
2. **클래스 위임**: 인터페이스를 구현할 때, 특정 메서드를 다른 객체에 위임할 수 있습니다.

위임을 사용하면 코드의 가독성이 높아지고, 변경 시의 리팩토링이 용이해집니다.

### 속성 위임 사용법
속성 위임을 사용하려면, `by` 키워드를 사용하여 속성에 위임할 객체를 지정합니다. 예를 들어, `Lazy` 위임을 사용할 수 있습니다.

### 클래스 위임 사용법
클래스 위임은 `by` 키워드를 사용하여 클래스가 특정 인터페이스의 메서드를 다른 클래스에 위임하는 방식입니다.

## 예제
### 속성 위임 예제
```kotlin
class User {
    var name: String by Delegates.observable("초기값") { property, oldValue, newValue ->
        println("$oldValue -> $newValue")
    }
}

fun main() {
    val user = User()
    user.name = "홍길동"
    user.name = "김철수"
}
```

### 클래스 위임 예제
```kotlin
interface Base {
    fun print()
}

class BaseImpl(val x: Int) : Base {
    override fun print() {
        println(x)
    }
}

class Derived(b: Base) : Base by b

fun main() {
    val b = BaseImpl(10)
    Derived(b).print()  // 출력: 10
}
```

## 설명
위임 기능을 사용할 때 주의해야 할 몇 가지 사항이 있습니다:

- **Null 안전성**: 위임 객체가 null일 경우, NullPointerException이 발생할 수 있습니다. 따라서 적절한 null 체크를 수행해야 합니다.
- **레이지 위임**: `Lazy` 위임을 사용할 경우, 속성에 접근할 때마다 초기화가 이루어지지 않도록 주의해야 합니다.
- **상속 관계**: 클래스를 위임할 경우, 상속 관계를 명확히 이해하고 있어야 합니다. 위임된 클래스의 메서드는 원래 클래스의 메서드를 덮어쓰게 됩니다.

## 한 줄 요약
코틀린의 위임(delegate)은 객체의 속성과 메서드를 다른 객체에 위임함으로써 코드의 재사용성과 유지보수성을 향상시키는 기능입니다.
