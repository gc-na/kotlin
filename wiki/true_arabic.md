# الكلمة "true" في كوتلن: فهم القيم المنطقية

## الملخص
تُعتبر الكلمة "true" في كوتلن واحدة من القيم المنطقية الأساسية، والتي تُستخدم بشكل شائع في التحكم في التدفق والتعبيرات الشرطية.

## الوثائق
### الغرض
تُستخدم الكلمة "true" في كوتلن لتمثيل القيمة المنطقية الصحيحة. تعتبر هذه القيمة جزءًا من نوع البيانات المنطقية (Boolean)، والذي يتضمن أيضًا القيمة "false".

### الاستخدام
تُستخدم "true" في التعبيرات الشرطية، الحلقات، والعمليات المنطقية. في كوتلن، يمكن استخدام هذه القيمة بشكل مباشر في الشروط أو كجزء من تعبيرات أكثر تعقيدًا.

### التفاصيل
- **نوع البيانات**: تعتبر "true" قيمة من نوع Boolean.
- **التعريف**: يتم استخدام "true" بشكل ثابت ولا يمكن تغيير قيمتها.
- **التعبيرات الشرطية**: تستخدم لتحديد مسارات تنفيذ معينة في البرمجة، مثل `if` و `when`.

## الأمثلة
### مثال 1: استخدام "true" في تعبير شرط
```kotlin
val isSunny = true

if (isSunny) {
    println("اليوم مشمس!")
} else {
    println("اليوم غائم.")
}
```

### مثال 2: استخدام "true" في حلقة تكرارية
```kotlin
var keepGoing = true
var count = 0

while (keepGoing) {
    count++
    if (count >= 5) {
        keepGoing = false
    }
}

println("تم التكرار $count مرات.")
```

## الشرح
### الأخطاء الشائعة
- **عدم استخدام القيم المنطقية بشكل صحيح**: يمكن أن يؤدي استخدام "true" في سياقات غير مناسبة إلى أخطاء منطقية.
- **الخلط بين "true" و "false"**: يجب التأكد من استخدام القيم المنطقية بشكل صحيح لتجنب الأخطاء البرمجية.

### ملاحظات إضافية
- من الجيد استخدام "true" كجزء من تعبيرات أكثر تعقيدًا، مثل استخدام الدوال التي تعيد القيم المنطقية.
- يجب مراعاة المقارنة بين القيم المنطقية للتأكد من صحة الشروط المستخدمة في الشيفرة.

## ملخص جملة واحدة
الكلمة "true" في كوتلن تُستخدم لتمثيل القيمة المنطقية الصحيحة وتعتبر أساسية في التحكم في تدفق البرنامج.
