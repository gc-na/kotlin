# الكلمة المفتاحية "super" في لغة كوتلن: الاستخدامات والأمثلة

## ملخص
كلمة "super" في كوتلن تُستخدم للإشارة إلى الفئات الأب (superclasses) واستدعاء خصائصها أو دوالها. تعتبر هذه الكلمة أساسية في مفهوم الوراثة في البرمجة كائنية التوجه.

## الوثائق
تُستخدم الكلمة المفتاحية "super" في كوتلن للوصول إلى الأعضاء (الخصائص والدوال) من الفئة الأب. تعتبر هذه الوظيفة ضرورية عند الحاجة لاستدعاء دوال أو خصائص تم تجاوزها في الفئة الابنة.

### الاستخدام
للاستخدام الصحيح لكلمة "super"، يمكن استخدامها على النحو التالي:

- **استدعاء دالة من الفئة الأب**: يمكن استخدام "super" لاستدعاء دالة من الفئة الأب، حتى لو كانت هذه الدالة مُعَاد تعريفها في الفئة الابنة.
- **الوصول إلى الخصائص**: يمكن أيضًا استخدام "super" للوصول إلى خصائص الفئة الأب التي قد تم تجاوزها في الفئة الابنة.

### التفاصيل
- يجب أن تكون الفئة الابنة مستخدمة في سياق يتطلب الوصول إلى خصائص أو دوال الفئة الأب.
- تُستخدم "super" فقط في السياقات التي تتطلب ذلك، مثل داخل دالة في الفئة الابنة.

## الأمثلة

### مثال 1: استدعاء دالة من الفئة الأب
```kotlin
open class Parent {
    open fun display() {
        println("Hello from Parent")
    }
}

class Child : Parent() {
    override fun display() {
        super.display() // استدعاء دالة الفئة الأب
        println("Hello from Child")
    }
}

fun main() {
    val child = Child()
    child.display()
}
```

### مثال 2: الوصول إلى خاصية من الفئة الأب
```kotlin
open class Parent {
    open val property: String = "Parent Property"
}

class Child : Parent() {
    override val property: String = "Child Property"

    fun showProperties() {
        println(super.property) // الوصول إلى خاصية الفئة الأب
    }
}

fun main() {
    val child = Child()
    child.showProperties()
}
```

## الشرح
من الأخطاء الشائعة عند استخدام "super" هو عدم استدعاء الدوال أو الخصائص بشكل صحيح، خاصة عندما يتم تجاوزها في الفئة الابنة. يجب على المطورين أن يكونوا على دراية بأن "super" تشير دائمًا إلى الفئة الأب التي يتم توارثها بشكل مباشر. 

قد يؤدي عدم فهم كيفية عمل "super" إلى أخطاء برمجية، خاصة عند التعامل مع الفئات المعقدة أو الفئات المتداخلة.

## ملخص جملة واحدة
تُستخدم الكلمة المفتاحية "super" في كوتلن للوصول إلى الخصائص والدوال من الفئة الأب، مما يسهل عملية الوراثة في البرمجة كائنية التوجه.
