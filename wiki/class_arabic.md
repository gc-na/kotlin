# الكلاس في كوتلن: مفهوم وإستخدامات

## الملخص
الكلاس في كوتلن هو بنية أساسية تُستخدم لتعريف كائنات (Objects) تحتوي على خصائص (Properties) ودوال (Functions) خاصة بها. تُعتبر الكلاسات ركيزة أساسية في البرمجة الكائنية (Object-Oriented Programming) في كوتلن.

## الوثائق
الكلاس في كوتلن يُستخدم لتجميع البيانات والسلوكيات المتعلقة بكائن معين. يتم تعريف الكلاس باستخدام الكلمة المفتاحية `class`. يمكن للكلاس أن يحتوي على خصائص، دوال، مُنشئات (Constructors)، ومُتغيرات. إليك الهيكل الأساسي لتعريف كلاس:

```kotlin
class ClassName {
    // الخصائص
    var propertyName: Type = value

    // الدوال
    fun functionName() {
        // منطق الدالة
    }
}
```

### الاستخدام
لتعريف كلاس في كوتلن، يمكنك بدء التعريف بالكلمة المفتاحية `class`، يليها اسم الكلاس، ثم تليها مجموعة الخصائص والدوال. يمكن أيضًا استخدام المُنشئات لتحديد كيفية إنشاء كائن من الكلاس.

## الأمثلة
### مثال أساسي
```kotlin
class Person(val name: String, var age: Int) {
    fun introduce() {
        println("مرحبًا، أنا $name وعُمري $age سنة.")
    }
}

fun main() {
    val person = Person("علي", 30)
    person.introduce()
}
```

### مثال مع مُنشئ مخصص
```kotlin
class Car(var model: String, var year: Int) {
    init {
        println("تم إنشاء سيارة جديدة: $model من سنة $year")
    }
}

fun main() {
    val car = Car("تويوتا", 2022)
}
```

## الشرح
عند العمل مع الكلاسات في كوتلن، من المهم أن تتذكر بعض النقاط:
- يمكن أن تحتوي الكلاسات على مُنشئات متعددة، بما في ذلك المُنشئات الافتراضية والمخصصة.
- يُمكن استخدام `init` لتشغيل الكود عند إنشاء كائن جديد.
- يجب تجنب استخدام الكلاسات كحاويات فارغة دون منطق أو خصائص، حيث أن الغرض من الكلاس هو تجميع البيانات والسلوكيات.

## ملخص جملة واحدة
الكلاس في كوتلن هو هيكل يُستخدم لتعريف كائنات تحتوي على خصائص ودوال، مما يسهل البرمجة الكائنية.
