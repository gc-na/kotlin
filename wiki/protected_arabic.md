# الحماية في كوتلن: فهم واستخدام "protected"

## ملخص
في كوتلن، تُستخدم كلمة "protected" للتحكم في وصول الأعضاء (الخصائص والطرق) في الفئات والفئات الفرعية. تساعد هذه الكلمة في تحقيق مستوى من الحماية لعناصر الفئة بحيث يمكن الوصول إليها فقط من الفئات الفرعية أو من نفس الفئة.

## الوثائق
تُعتبر كلمة "protected" واحدة من محددات الوصول في كوتلن. عند استخدام "protected" مع خصائص أو طرق، يكون الوصول إليها محصورًا بالفئات الفرعية فقط. هذا يعني أن الأعضاء المحمية لا يمكن الوصول إليها من خارج الفئة أو الفئات الفرعية، مما يُعتبر طريقة فعالة للحفاظ على سلامة البيانات وتقليل التعقيد.

### الاستخدام
لإعلان خاصية أو دالة محمية، يتم استخدام الكلمة "protected" قبل تعريفها. إليك الصيغة الأساسية:

```kotlin
protected val propertyName: Type
protected fun functionName() { ... }
```

### التفاصيل
- **الفئات الفرعية**: يمكن للفئات الفرعية الوصول إلى الأعضاء المحمية، مما يجعلها مثالية للاستخدام في تصميم الأنظمة المعقدة حيث يمكن توسيع الفئات بسهولة.
- **الفئات الداخلية**: يمكن أيضًا للفئات الداخلية أن تصل إلى الأعضاء المحمية، مما يوفر مزيدًا من المرونة.
- **الوصول من الكود الخارجي**: لا يمكن الوصول إلى الأعضاء المحمية من خارج الفئة أو الفئة الفرعية، مما يعزز الأمان.

## الأمثلة
إليك بعض الأمثلة على كيفية استخدام "protected" في كوتلن:

### مثال 1: استخدام الخصائص المحمية
```kotlin
open class Parent {
    protected val secret = "هذا سر"
}

class Child : Parent() {
    fun revealSecret() {
        println(secret) // يمكن الوصول إلى 'secret' هنا
    }
}
```

### مثال 2: استخدام الدوال المحمية
```kotlin
open class Animal {
    protected fun makeSound() {
        println("صوت حيوان")
    }
}

class Dog : Animal() {
    fun bark() {
        makeSound() // يمكن الوصول إلى 'makeSound' هنا
        println("نباح")
    }
}
```

## الشرح
قد يواجه المبرمجون بعض التحديات عند استخدام "protected"، مثل:

- **عدم القدرة على الوصول من الفئات غير المرتبطة**: إذا حاولت الوصول إلى خاصية محمية من فئة غير فرعية، ستحصل على خطأ في الكود.
- **تصميم معقد**: استخدام "protected" بشكل مفرط قد يؤدي إلى تعقيد التصميم، لذا يجب استخدامه بحذر.

## ملخص بجملة واحدة
تتيح "protected" في كوتلن التحكم في وصول الأعضاء في الفئات والفئات الفرعية، مما يعزز الحماية ويسهل إدارة البيانات.
