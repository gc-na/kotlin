# استخدام الكلمة المفتاحية "override" في لغة كوتلن

## الملخص
تعتبر الكلمة المفتاحية "override" في كوتلن عنصرًا أساسيًا في البرمجة الكائنية، حيث تُستخدم لتجاوز الدوال أو الخصائص الموروثة من الفئات الأب.

## الوثائق
### الغرض
تُستخدم "override" في كوتلن لتحديد أن دالة أو خاصية معينة في فئة فرعية تعيد تعريف (تتجاوز) دالة أو خاصية بنفس الاسم في الفئة الأم. يسمح هذا المبدأ بتخصيص سلوك الكائنات المشتقة.

### الاستخدام
عند تعريف دالة أو خاصية في فئة فرعية، يجب أن تكون الدالة أو الخاصية في الفئة الأم مُعلمة بالكلمة المفتاحية "open" لتسمح بتجاوزها. تتم كتابة الكلمة المفتاحية "override" قبل تعريف الدالة أو الخاصية في الفئة الفرعية.

### التفاصيل
- **الدوال**: يمكن تجاوز أي دالة مُعلمة بـ "open" في الفئة الأم.
- **الخصائص**: يمكن تجاوز الخصائص بنفس الطريقة.
- **المتطلبات**: يجب أن تكون الفئة الأم مفتوحة للتوريث باستخدام الكلمة المفتاحية "open".

## أمثلة
### مثال على تجاوز دالة
```kotlin
open class Animal {
    open fun sound() {
        println("Some sound")
    }
}

class Dog : Animal() {
    override fun sound() {
        println("Bark")
    }
}

fun main() {
    val dog = Dog()
    dog.sound()  // Output: Bark
}
```

### مثال على تجاوز خاصية
```kotlin
open class Vehicle {
    open val type: String = "Generic Vehicle"
}

class Car : Vehicle() {
    override val type: String = "Car"
}

fun main() {
    val myCar = Car()
    println(myCar.type)  // Output: Car
}
```

## الشرح
- **الأخطاء الشائعة**: 
  - عدم استخدام "open" في الفئة الأم مما يؤدي إلى عدم إمكانية تجاوز الدالة أو الخاصية.
  - نسيان استخدام "override" عند تعريف الدالة أو الخاصية في الفئة الفرعية.
  
- **ملاحظات إضافية**:
  - يمكن استخدام الكلمة المفتاحية "final" في الفئة الأم أو في الدالة أو الخاصية لمنع تجاوزها.
  - يُفضل الحفاظ على تنظيم الكود واستخدام "override" عند الحاجة لضمان وضوح الكود وسهولة الصيانة.

## ملخص جملة واحدة
تُستخدم الكلمة المفتاحية "override" في كوتلن لتجاوز الدوال أو الخصائص الموروثة من الفئات الأب، مما يوفر إمكانية تخصيص سلوك الكائنات المشتقة.
