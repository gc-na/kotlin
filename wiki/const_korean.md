# Kotlin의 const: 상수 정의와 활용

## 개요
Kotlin에서 `const`는 컴파일 타임에 상수를 정의하는 데 사용되는 키워드로, 주로 상수값을 사용해야 할 때 유용합니다. 이 키워드는 읽기 전용 프로퍼티를 정의하는 데 사용되며, 상수 값이 프로그램 실행 중 변경되지 않음을 보장합니다.

## 문서화
`const`는 Kotlin의 최상위 프로퍼티 또는 객체의 프로퍼티로 사용될 수 있으며, 항상 `val`과 함께 사용됩니다. `const`로 정의된 프로퍼티는 다음과 같은 특징을 가집니다:

- **컴파일 타임 상수**: `const`로 선언된 값은 컴파일 시점에 결정되며, 런타임에 변경할 수 없습니다.
- **정수, 문자열, 불리언**: `const`는 기본 데이터 타입인 정수형, 문자열, 불리언형에만 적용할 수 있습니다.
- **전역 사용 가능**: `const` 프로퍼티는 최상위 레벨 또는 객체 내부에서 정의할 수 있습니다.

사용 예:
```kotlin
const val MAX_USER_COUNT = 100
const val API_URL = "https://api.example.com"
```

이러한 방식으로 `const`를 사용하면 코드의 가독성을 높이고, 하드코딩된 값을 피할 수 있습니다.

## 예제
다음은 `const` 키워드를 사용한 기본적인 예제입니다.

```kotlin
const val PI = 3.14

fun calculateArea(radius: Double): Double {
    return PI * radius * radius
}

fun main() {
    val area = calculateArea(5.0)
    println("원의 면적: $area")
}
```

위의 예제에서 `PI`는 원의 면적을 계산하는 데 사용되는 상수로, 변경될 수 없는 값입니다. 

## 설명
`const`를 사용할 때 주의할 점은 다음과 같습니다:

- **타입 제한**: `const`로 선언할 수 있는 데이터 타입은 제한적입니다. 기본적인 데이터 타입(정수, 문자열, 불리언)만 가능하므로 사용자 정의 타입이나 복잡한 데이터 구조에는 사용할 수 없습니다.
- **초기화 규칙**: `const`로 선언된 프로퍼티는 반드시 선언과 동시에 초기화해야 하며, 실행 중에 값을 변경할 수 없습니다.
- **전역 사용**: 최상위 레벨에서 정의된 `const` 프로퍼티는 코드 전반에서 접근 가능하므로, 잘못된 사용이 발생할 수 있습니다. 따라서 명확한 네이밍과 주석을 통해 코드의 의도를 분명히 하는 것이 좋습니다.

## 한 줄 요약
Kotlin에서 `const`는 컴파일 타임에 결정되는 읽기 전용 상수를 정의하기 위해 사용되는 키워드입니다.
