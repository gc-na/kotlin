# Kotlin 中的 val：不可變變數的定義與用法

## 摘要
在 Kotlin 中，`val` 是用來宣告不可變變數的關鍵字，這意味著一旦賦值後，變數的值將無法被重新賦值。這種特性使得 Kotlin 在編程時更加安全和可預測。

## 文件說明
`val` 是 Kotlin 語言中用來定義只讀變數的關鍵字。與 `var` 不同，使用 `val` 定義的變數一旦被賦值後將不能再修改。這使得代碼在設計上更具可讀性和健壯性，降低了出錯的可能性。

### 用法
在 Kotlin 中，使用 `val` 來定義變數的基本語法如下：
```kotlin
val variableName: Type = value
```
- **variableName**：變數名稱
- **Type**：變數類型（可選，Kotlin 可以自動推斷）
- **value**：賦予變數的值

### 詳細說明
- 當你使用 `val` 定義變數時，該變數的值在初始化後不可修改，這使得代碼更加清晰和安全。
- `val` 定義的變數可以是任何類型，包括基本數據類型、集合、對象等。
- 如果你需要一個可以在後續修改的變數，則應使用 `var`。

## 範例
以下是使用 `val` 的基本範例：

```kotlin
val name: String = "Kotlin"
val number = 42 // 自動推斷為 Int

// 嘗試修改變數會導致編譯錯誤
// name = "Java" // 錯誤：無法重新賦值
```

## 解釋
- **常見陷阱**：初學者可能會誤解 `val` 的性質，認為它使變數的內容不可變，實際上，`val` 只是防止變數本身的重新賦值。如果變數指向一個可變的對象，則對象的內容仍然可以被修改。
  
  ```kotlin
  val list = mutableListOf(1, 2, 3)
  list.add(4) // 這是合法的，因為 list 的內容可以改變
  ```

- **其他注意事項**：在使用 `val` 時，務必確保變數的初始值正確，因為之後將不能再更改。

## 一行總結
在 Kotlin 中，`val` 用於定義不可變變數，提供了安全性和可讀性。
