# Kotlin의 init: 생성자 초기화

## 개요
Kotlin의 `init` 블록은 클래스의 주 생성자와 함께 사용되어 객체가 생성될 때 초기화 작업을 수행하는 데 사용됩니다. 이 블록은 클래스의 인스턴스가 생성될 때 자동으로 호출되어, 필요한 초기값 설정이나 로직을 구현할 수 있습니다.

## 문서화
`init` 블록은 Kotlin 클래스의 주 생성자와 함께 명시적으로 정의됩니다. 이 블록은 클래스의 인스턴스가 생성될 때 자동으로 실행되며, 주 생성자의 매개변수를 활용하여 객체의 초기 상태를 설정할 수 있습니다.

### 용도
- 클래스의 프로퍼티 초기화
- 복잡한 초기화 로직 수행
- 필수적인 상태 검증

### 사용법
`init` 블록은 클래스 내부에서 `init` 키워드를 사용하여 정의합니다. 주 생성자와 함께 사용되며, 여러 개의 `init` 블록을 가질 수 있습니다. 각각의 `init` 블록은 인스턴스가 생성될 때 순서대로 실행됩니다.

## 예제
```kotlin
class Person(val name: String, val age: Int) {
    init {
        println("이름: $name, 나이: $age")
    }
}

fun main() {
    val person = Person("홍길동", 30)
}
```
위의 예제에서는 `Person` 클래스의 인스턴스가 생성될 때 `init` 블록이 호출되어, 이름과 나이를 출력합니다.

## 설명
- `init` 블록은 필드를 초기화하는 데 유용하지만, 클래스의 인스턴스가 생성되는 과정에서 예외가 발생할 수 있습니다. 예를 들어, 필수 값이 누락되었거나 유효하지 않은 경우 초기화 로직에서 예외가 발생할 수 있습니다.
- `init` 블록 내에서 주의해야 할 점은 클래스의 프로퍼티가 초기화되기 전에 접근할 수 없다는 것입니다. 주 생성자에서 초기화되지 않은 프로퍼티에 접근하면 컴파일 오류가 발생합니다.

## 한 줄 요약
Kotlin의 `init` 블록은 클래스의 인스턴스가 생성될 때 초기화 작업을 수행하는 키워드입니다.
