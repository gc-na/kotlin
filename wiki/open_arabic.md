# الكلمة المفتاحية "open" في كوتلن: كل ما تحتاج معرفته

## ملخص
تُستخدم الكلمة المفتاحية "open" في كوتلن لتحديد الصفوف (classes) والدوال (functions) التي يمكن أن تُورَّث منها. على عكس لغات البرمجة الأخرى، حيث تكون الصفوف مُغلقة بشكل افتراضي، فإن كوتلن يتطلب استخدام "open" للسماح بالوراثة.

## الوثائق
### الغرض
تساعد الكلمة المفتاحية "open" على تحديد الصفوف والدوال التي يمكن أن تُمدد (extend) أو تُعدل (override) في صفوف أخرى. هذا يجعل كوتلن لغة مرنة تسمح بإنشاء هياكل برمجية معقدة بسهولة.

### الاستخدام
- **الصفوف**: يمكن استخدام "open" قبل تعريف الصف لتحديد أنه قابل للتوريث.
- **الدوال**: يمكن استخدام "open" قبل تعريف دالة داخل صف مفتوح للسماح بتعديلها في الصفوف المشتقة.

### التفاصيل
في كوتلن، إذا لم يتم تحديد الصف أو الدالة كـ "open"، فلن يمكن استخدامهما للتوريث أو التعديل. هذا يجعل كوتلن أكثر أمانًا ويقلل من الأخطاء المحتملة الناتجة عن الوراثة غير المتوقعة.

## الأمثلة
### مثال على الصف المفتوح
```kotlin
open class Animal {
    open fun sound() {
        println("Some sound")
    }
}

class Dog : Animal() {
    override fun sound() {
        println("Bark")
    }
}
```
في هذا المثال، يتم تعريف الصف `Animal` كـ "open"، مما يسمح بإنشاء صف `Dog` الذي يورث منه ويعدل دالة `sound`.

### مثال على دالة مفتوحة
```kotlin
open class Vehicle {
    open fun start() {
        println("Vehicle starting")
    }
}

class Car : Vehicle() {
    override fun start() {
        println("Car starting")
    }
}
```
هنا، تم تعريف الدالة `start` كـ "open"، مما يسمح لـ `Car` بتعديل سلوك الدالة.

## الشرح
### المشكلات الشائعة
- **عدم استخدام "open"**: إذا نسيت إضافة الكلمة "open" لصف أو دالة، فلن تتمكن من توريثها أو تعديلها.
- **التوريث المتعدد**: كوتلن لا يدعم التوريث المتعدد، لذا يجب توخي الحذر عند تصميم الصفوف.

### ملاحظات إضافية
- استخدام "open" في الصفوف والدوال يمكن أن يساعد في تحسين قابلية إعادة الاستخدام في الكود.
- يُفضل دائمًا استخدام "final" لتحديد الصفوف والدوال التي لا ينبغي توريثها، مما يحسن من أداء التطبيق.

## ملخص جملة واحدة
تُستخدم الكلمة المفتاحية "open" في كوتلن لتحديد الصفوف والدوال القابلة للتوريث، مما يوفر مرونة أكبر في تصميم البرمجيات.
