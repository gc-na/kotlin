# استخدام "infix" في لغة كوتلن: دليلك الشامل

## الملخص
في كوتلن، يُستخدم "infix" لتعريف دوال خاصة تسمح باستخدامها بطريقة أكثر قراءة ووضوحًا، مما يسهل كتابة التعبيرات الرياضية أو المنطقية.

## التوثيق
### الغرض
يهدف "infix" إلى تحسين وضوح الكود من خلال السماح بكتابة دوال بطريقة تشبه التعبيرات الطبيعية. يسمح هذا للمطورين باستخدام الدوال بشكل أكثر سلاسة، مما يجعل الكود أكثر سهولة في الفهم.

### الاستخدام
لتعريف دالة كـ "infix"، يجب أن تكون دالة عضو (member function) في فئة ما، ويجب أن تأخذ معلمة واحدة فقط. يمكن استدعاء هذه الدالة باستخدام صيغة infix مباشرة بين الكائنات، مما يلغي الحاجة لاستخدام الأقواس أو النقاط.

### التفاصيل
- يجب أن تكون الدالة تابعة (member function) لفئة أو كائن.
- يمكن استخدام "infix" فقط مع الدوال التي تأخذ معلمة واحدة.
- لا يمكن استخدام "infix" مع الدوال التي تُعرف كـ "extension functions".

## أمثلة
### تعريف دالة باستخدام "infix"
```kotlin
class Number(val value: Int) {
    infix fun add(other: Number): Number {
        return Number(this.value + other.value)
    }
}

fun main() {
    val num1 = Number(5)
    val num2 = Number(10)
    val result = num1 add num2
    println(result.value) // الناتج: 15
}
```

### استخدام "infix" مع دوال متعددة
```kotlin
class Person(val name: String) {
    infix fun isFriendOf(other: Person): String {
        return "$name is a friend of ${other.name}"
    }
}

fun main() {
    val alice = Person("Alice")
    val bob = Person("Bob")
    println(alice isFriendOf bob) // الناتج: Alice is a friend of Bob
}
```

## الشرح
### مشكلات شائعة
- يجب أن تتذكر أن "infix" لا يمكن استخدامها مع الدوال التي تأخذ أكثر من معلمة.
- قد يؤدي استخدام "infix" مع أسماء دوال غير واضحة أو غير دالة إلى صعوبة في فهم الكود.
- تأكد من استخدام "infix" فقط عندما يكون ذلك منطقيًا من حيث القراءة، حيث قد تصبح بعض التعبيرات معقدة أو مربكة.
