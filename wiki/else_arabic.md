# else في لغة كوتلن: كل ما تحتاج معرفته

## ملخص
تُستخدم عبارة "else" في لغة كوتلن كجزء من العبارات الشرطية لتوفير مسار بديل للتنفيذ عندما لا تتحقق الشروط المحددة في عبارة "if". تعتبر عبارة "else" أداة هامة لتحسين تدفق التحكم في البرمجة.

## الوثائق
عبارة "else" تُستخدم مع عبارات "if" لتحديد الكود الذي يجب تنفيذه في حالة عدم تحقق الشرط. يُمكن أن تتواجد عبارة "else" بمفردها أو تُستخدم مع عبارة "if" في شكل عبارة "if-else". 

### الاستخدام
يمكن استخدامها بالشكل التالي:

```kotlin
if (شرط) {
    // تنفيذ الكود إذا تحقق الشرط
} else {
    // تنفيذ الكود إذا لم يتحقق الشرط
}
```

### التفاصيل
- **التعشيش**: يمكن استخدام "else" مع "if" في تعشيش، مما يتيح لك إنشاء العديد من المسارات الشرطية. 
- **الأنواع**: يمكن استخدام "else" مع أي نوع من البيانات، بما في ذلك الأعداد الصحيحة، السلاسل، القيم المنطقية، وغيرها.
- **عدم وجود شرط**: يمكن أن تتواجد عبارة "else" بدون عبارة "if" مسبقة، ولكن في هذه الحالة يجب أن يكون هناك شرط كافٍ لتحديد ما يجب القيام به.

## أمثلة
### مثال 1: استخدام بسيط لعبارة "else"
```kotlin
val العدد = 10
if (العدد > 5) {
    println("العدد أكبر من 5")
} else {
    println("العدد ليس أكبر من 5")
}
```

### مثال 2: تعشيش "if-else"
```kotlin
val درجة = 75
if (درجة >= 90) {
    println("ممتاز")
} else if (درجة >= 75) {
    println("جيد جداً")
} else {
    println("يجب التحسين")
}
```

## الشرح
### الأخطاء الشائعة
- **نسيان استخدام الأقواس**: يجب التأكد من استخدام الأقواس بشكل صحيح لتحديد نطاق الكود الذي يتبع العبارة الشرطية.
- **عدم استخدام "else" بعد "if"**: من الممكن أن يؤدي ذلك إلى عدم تنفيذ الكود كما هو متوقع، حيث سيتم تجاهل الكود الذي يجب تنفيذه إذا لم يتحقق الشرط.

### ملاحظات إضافية
- يُمكن استخدام تعبير "else" مع "when" في كوتلن كبديل عن استخدام "if-else" في بعض الحالات، مما يوفر طريقة أكثر وضوحاً لترتيب الشروط.
  
## ملخص بنغمة واحدة
عبارة "else" في كوتلن تُستخدم لتحديد الكود البديل للتنفيذ عند عدم تحقق شرط "if".
