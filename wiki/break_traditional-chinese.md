# Kotlin中的「break」語句：用於控制迴圈的關鍵字

## 摘要
在Kotlin編程語言中，「break」語句用於提前終止迴圈的執行，無論是`for`、`while`還是`do...while`迴圈。這個特性使得在特定條件下能夠更靈活地控制程式的流向。

## 文檔
「break」是一個控制流語句，主要用於中斷迴圈的執行。當執行到「break」語句時，控制權會立即跳出當前的迴圈，並繼續執行迴圈後的程式碼。

### 用法
「break」的基本語法形式如下：

```kotlin
break
```

當「break」語句被執行時，將結束最近的外層迴圈。如果有嵌套的迴圈，則只有最內層的迴圈會被終止。

## 範例
以下是「break」語句的基本使用範例：

### 範例 1：終止`for`迴圈
```kotlin
fun main() {
    for (i in 1..10) {
        if (i == 5) {
            break // 當i等於5時，終止迴圈
        }
        println(i)
    }
}
```
輸出：
```
1
2
3
4
```

### 範例 2：終止`while`迴圈
```kotlin
fun main() {
    var i = 1
    while (i <= 10) {
        if (i == 7) {
            break // 當i等於7時，終止迴圈
        }
        println(i)
        i++
    }
}
```
輸出：
```
1
2
3
4
5
6
```

## 解釋
使用「break」語句時，有幾個常見的注意事項：

1. **控制流的清晰性**：過度使用「break」可能使程式的控制流變得不易理解。應該謹慎使用，以保持程式碼的可讀性。
2. **嵌套迴圈**：在嵌套的迴圈中，「break」只會終止最內層的迴圈。如果需要終止外層迴圈，可以使用標籤（label）來實現更精確的控制。
3. **與「continue」的區別**：與「break」相對的是「continue」，後者會跳過當前迴圈的剩餘部分，並轉到下一次迴圈的執行。

## 一句總結
「break」語句在Kotlin中用於提前終止迴圈的執行，提升程式的控制靈活性。
