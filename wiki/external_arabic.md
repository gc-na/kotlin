# الكلمة المفتاحية "external" في كوتلن: الاستخدامات والتفاصيل

## ملخص
تُستخدم الكلمة المفتاحية "external" في كوتلن للإشارة إلى أن الدالة أو المتغير مُعرف في مكان آخر، مثل مكتبات خارجية أو ملفات أخرى. تعتبر هذه الكلمة أساسية عند العمل مع كود مكتوب بلغة أخرى، مثل C أو C++، حيث تتيح لك استدعاء الدوال والخصائص من تلك اللغات.

## الوثائق
### الغرض
تُستخدم الكلمة المفتاحية "external" في كوتلن لتعريف دوال أو خصائص تُنفذ خارج نطاق البرنامج الحالي، مما يسهل التكامل مع كود مكتوب بلغة أخرى.

### الاستخدام
يمكنك تعريف دالة أو خاصية خارجية باستخدام الكلمة المفتاحية "external" كما يلي:

```kotlin
external fun someFunction(): String
external val someVariable: Int
```

### التفاصيل
- **الربط مع المكتبات الخارجية**: يمكنك استخدام "external" لربط كود كوتلن بمكتبات مكتوبة بلغة C أو C++.
- **التوافق مع NDK**: عند تطوير تطبيقات أندرويد، تعتبر "external" مفيدة جداً عند استخدام NDK للتفاعل مع كود C/C++.
- **تحديد موقع التنفيذ**: يجب أن يكون لديك رمز مكتوب بلغة غير كوتلن يتوافق مع التعريف الذي تقدمه.

## أمثلة
### مثال 1: تعريف دالة خارجية
```kotlin
external fun nativeMethod(): String
```
### مثال 2: تعريف متغير خارجي
```kotlin
external val nativeVariable: Int
```
### مثال 3: استخدام الدالة الخارجية
```kotlin
fun main() {
    println(nativeMethod())
}
```

## الشرح
### الأخطاء الشائعة
- **الإخفاق في الربط**: تأكد من أن المكتبة الخارجية موجودة ومربوطة بشكل صحيح، حيث أن عدم وجود المكتبة سيؤدي إلى أخطاء في وقت التشغيل.
- **تسمية غير دقيقة**: تأكد من أن أسماء الدوال أو المتغيرات تتطابق تمامًا مع تلك المعرفة في المكتبة الخارجية.
- **التوافق مع النظام**: تحقق من توافق المكتبة مع النظام الذي تقوم بتشغيل التطبيق عليه، فقد تختلف المكتبات بين أنظمة التشغيل.

### ملاحظات إضافية
- استخدم "external" فقط عندما تحتاج إلى استدعاء كود مكتوب بلغة أخرى. 
- تأكد من قراءة الوثائق الخاصة بالمكتبة الخارجية المستخدمة لفهم كيفية استخدامها بشكل صحيح.

## ملخص جملة واحدة
تستخدم الكلمة المفتاحية "external" في كوتلن لتعريف دوال وخصائص مُعرفة في مكتبات أو ملفات خارجية، مما يسهل التكامل مع كود مكتوب بلغات أخرى.
