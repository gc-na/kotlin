# استخدام "try" في لغة كوتلن: التعامل مع الاستثناءات بشكل فعال

## ملخص
تُستخدم عبارة "try" في لغة كوتلن للتعامل مع الاستثناءات، مما يتيح للمطورين التعامل مع الأخطاء المحتملة في البرنامج بطريقة منظمة.

## الوثائق
عبارة "try" في كوتلن هي جزء من آلية معالجة الاستثناءات التي تساعد على تحديد الكود الذي قد ينتج عنه استثناء، مما يسمح بالتعامل معه بشكل مناسب. يتم استخدام "try" لاحتواء الكود الذي قد يُسبب استثناء، ويُتبع عادةً بـ "catch" لالتقاط الاستثناءات ومعالجتها. يمكن أيضًا استخدام "finally" لضمان تنفيذ كود معين بعد محاولة المعالجة.

### الغرض
الغرض من استخدام "try" هو تحسين استقرار البرامج من خلال السماح للمطورين بالتعامل مع الأخطاء بشكل أكثر فعالية.

### الاستخدام
تُستخدم "try" بالشكل التالي:
```kotlin
try {
    // الكود الذي قد يُسبب استثناء
} catch (e: ExceptionType) {
    // معالجة الاستثناء
} finally {
    // كود سيتم تنفيذه دائمًا
}
```

## الأمثلة
### مثال 1: استخدام "try" مع "catch"
```kotlin
fun main() {
    try {
        val number = 10 / 0 // محاولة قسمة على صفر
    } catch (e: ArithmeticException) {
        println("حدث استثناء: ${e.message}")
    }
}
```
### مثال 2: استخدام "try" مع "finally"
```kotlin
fun main() {
    try {
        val file = java.io.File("non_existent_file.txt")
        val reader = java.io.BufferedReader(java.io.FileReader(file))
        println(reader.readLine())
    } catch (e: FileNotFoundException) {
        println("الملف غير موجود: ${e.message}")
    } finally {
        println("تم الانتهاء من محاولة القراءة.")
    }
}
```

## الشرح
### الأخطاء الشائعة
- **تجاهل الاستثناءات**: يمكن أن يؤدي عدم التعامل مع الاستثناءات بشكل صحيح إلى تعطل البرنامج.
- **عدم استخدام "finally" عند الحاجة**: قد يكون من الضروري دائمًا تنفيذ كود معين مثل تحرير الموارد.

### ملاحظات إضافية
- يمكن استخدام جملة "try" مع أنواع متعددة من الاستثناءات.
- يمكن أن تحتوي جملة "catch" على عدة معالجات لاستثناءات مختلفة.
- يُفضل دائمًا تحديد الاستثناءات بدقة بدلاً من استخدام "Exception" العامة لتجنب إخفاء الأخطاء.

## ملخص في جملة واحدة
عبارة "try" في كوتلن تُستخدم للتعامل مع الاستثناءات، مما يسمح بتحسين استقرار البرنامج من خلال معالجة الأخطاء بشكل منظم.
