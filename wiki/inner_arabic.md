# الكلمة المحجوزة "inner" في كوتلن: الفئات الداخلية

## الملخص
تُستخدم الكلمة المحجوزة "inner" في كوتلن لتعريف الفئات الداخلية، وهي فئات تُعَرَّف داخل فئة أخرى. تسمح الفئات الداخلية بالوصول إلى الأعضاء (الخصائص والطرق) للفئة الخارجية.

## الوثائق
### الغرض
تُستخدم الفئات الداخلية لتوفير علاقة أقرب بين الفئة الداخلية والفئة الخارجية. يمكن للفئات الداخلية الوصول إلى الأعضاء الخاصة والعامة للفئة الخارجية، مما يسهل تنظيم الكود وتبسيط الوصول إلى البيانات.

### الاستخدام
لتعريف فئة داخلية، يتم استخدام الكلمة المحجوزة "inner" قبل تعريف الفئة. هنا مثال على كيفية تعريف فئة داخلية:

```kotlin
class Outer {
    private val outerProperty = "هذا خاص بفئة Outer"

    inner class Inner {
        fun accessOuterProperty() = outerProperty
    }
}
```

### التفاصيل
- **الوصول إلى الأعضاء:** الفئات الداخلية يمكنها الوصول إلى الخصائص والطرق الخاصة بالفئة الخارجية.
- **تداخل الفئات:** يمكن تعريف عدة فئات داخلية داخل فئة خارجية، مما يسمح بإنشاء هياكل بيانات معقدة.
- **تسميات الفئات:** الفئة الداخلية يمكن أن تستخدم نفس أسماء الفئات الخارجية، ولكن يجب أن تكون متميزة عن باقي الفئات الداخلية.

## الأمثلة
### مثال 1: استخدام الفئات الداخلية
```kotlin
class Car {
    private val brand = "تويوتا"

    inner class Engine {
        fun getBrand() = brand
    }
}

fun main() {
    val car = Car()
    val engine = car.Engine()
    println(engine.getBrand())  // يطبع "تويوتا"
}
```

### مثال 2: عدة فئات داخلية
```kotlin
class House {
    inner class Room {
        fun describe() = "هذه غرفة داخل المنزل."
    }

    inner class Bathroom {
        fun describe() = "هذه حمام داخل المنزل."
    }
}

fun main() {
    val house = House()
    val room = house.Room()
    val bathroom = house.Bathroom()
    println(room.describe())     // يطبع "هذه غرفة داخل المنزل."
    println(bathroom.describe()) // يطبع "هذه حمام داخل المنزل."
}
```

## الشرح
### الأخطاء الشائعة
- **عدم استخدام الكلمة المحجوزة "inner":** إذا لم يتم استخدام الكلمة "inner"، فإن الفئة الداخلية لن تتمكن من الوصول إلى الأعضاء الخاصة للفئة الخارجية.
- **استخدام الفئات الداخلية بشكل غير ضروري:** ينبغي استخدام الفئات الداخلية فقط عندما يكون ذلك منطقيًا من حيث التصميم، لتفادي التعقيد غير الضروري.

### ملاحظات إضافية
- يمكن أن تؤدي الفئات الداخلية إلى زيادة التعقيد في الكود إذا تم استخدامها بشكل مفرط، لذا يجب استخدامها بحذر.

## ملخص جملة واحدة
تتيح الكلمة المحجوزة "inner" في كوتلن تعريف الفئات الداخلية التي يمكنها الوصول إلى الأعضاء الخاصة للفئة الخارجية، مما يسهل تنظيم الكود.
