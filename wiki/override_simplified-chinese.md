# Kotlin中的“override”关键字详解

## 概述
在Kotlin中，“override”关键字用于标记一个方法或属性是从父类或接口中重写的。这使得子类可以对继承自父类的行为进行定制和扩展。

## 文档
### 目的
“override”关键字的主要目的是在继承层次结构中，通过重写父类的方法或属性，提供特定于子类的实现。这一特性是面向对象编程的重要组成部分，使得代码复用和扩展变得更加灵活。

### 用法
在Kotlin中，当子类需要重写父类的方法时，必须在父类的方法声明前加上“open”关键字，表示该方法可以被重写。接着，在子类中使用“override”关键字进行重写。

### 详细说明
- **父类定义**: 在Kotlin中，类默认是封闭的，意味着无法被继承。要允许类被继承，必须使用“open”关键字。
- **重写方法**: 子类使用“override”关键字来重写父类的方法。
- **重写属性**: 同样可以重写属性，使用“override”关键字来实现。

## 示例
以下是一个简单的示例，演示如何使用“override”关键字：

```kotlin
open class Animal {
    open fun sound() {
        println("动物发出声音")
    }
}

class Dog : Animal() {
    override fun sound() {
        println("汪汪")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // 输出: 汪汪
}
```

在上面的代码中，`Animal`类的`sound`方法被`Dog`类重写，从而提供了一个特定的实现。

## 解释
### 常见问题
- **未找到override**: 如果试图重写一个未标记为“open”的父类方法，Kotlin编译器将报错，提示该方法不能被重写。
- **重写多个接口**: 当实现多个接口时，如果不同接口中存在同名方法，子类必须明确重写这些方法以消除歧义。
- **父类调用**: 在重写的方法中，可以使用`super`关键字调用父类的方法。

### 注意事项
- 确保父类的方法被标记为“open”。
- 使用“override”时，方法签名必须与父类完全匹配（包括参数类型和返回类型）。

## 一句话总结
在Kotlin中，使用“override”关键字可以重写父类的方法或属性，从而实现子类特定的行为。
