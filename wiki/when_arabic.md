# استخدام "when" في كوتلن: دليل شامل

## ملخص
يعتبر "when" في كوتلن تعبيرًا قويًا للتحكم في التدفق، حيث يُستخدم كبديل لجملة `switch` في لغات البرمجة الأخرى. يتيح لك تحديد مجموعة من الشروط وتنفيذ كود مختلف بناءً على القيمة المدخلة.

## الوثائق
### الغرض
تُستخدم عبارة "when" لتحديد تدفق البرنامج بناءً على القيم المختلفة. إنها مفيدة بشكل خاص عند الحاجة إلى مقارنة متغير مع مجموعة من القيم المحتملة.

### الاستخدام
تستطيع استخدام "when" مع أي نوع من الأنواع (مثل الأعداد الصحيحة، السلاسل النصية، إلخ). يمكن أن تحتوي على تعبيرات متعددة، مما يجعلها أكثر مرونة مقارنةً بـ `if` في بعض السيناريوهات.

### التفاصيل
- **الصيغة الأساسية**:
    ```kotlin
    when (variable) {
        value1 -> { /* تنفيذ كود */ }
        value2 -> { /* تنفيذ كود */ }
        else -> { /* تنفيذ كود إذا لم تنطبق أي من القيم السابقة */ }
    }
    ```

- **استخدامات متقدمة**:
    - يمكن استخدام "when" بدون متغير، حيث يمكن أن يكون تعبيرًا شرطياً:
      ```kotlin
      when {
          condition1 -> { /* تنفيذ كود */ }
          condition2 -> { /* تنفيذ كود */ }
          else -> { /* تنفيذ كود */ }
      }
      ```

## الأمثلة
### مثال 1: استخدام "when" مع القيم الثابتة
```kotlin
val number = 3
when (number) {
    1 -> println("الرقم واحد")
    2 -> println("الرقم اثنان")
    3 -> println("الرقم ثلاثة")
    else -> println("رقم غير معروف")
}
```

### مثال 2: استخدام "when" كشرط
```kotlin
val score = 85
when {
    score >= 90 -> println("ممتاز")
    score >= 75 -> println("جيد جداً")
    score >= 60 -> println("جيد")
    else -> println("راسب")
}
```

## الشرح
### الأخطاء الشائعة
- **نسيان حالة `else`**: إذا لم يتم تضمين حالة `else`، قد يؤدي ذلك إلى خطأ في وقت التشغيل إذا لم تتطابق أي من القيم.
- **عدم استخدام الأقواس**: تأكد من استخدام الأقواس `{}` عند كتابة كتلة من التعليمات البرمجية، وإلا قد تكون النتيجة غير متوقعة.

### ملاحظات إضافية
- يمكن استخدام "when" مع أنواع البيانات المختلفة، بما في ذلك الأنواع المخصصة.
- "when" يمكن أن يكون مفيدًا في تجنب تكرار الشيفرة، مما يجعل الكود أكثر نظافة.

## ملخص جملة واحدة
عبارة "when" في كوتلن هي تعبير قوي ومرن للتحكم في تدفق البرنامج بناءً على مجموعة من الشروط والقيم المحتملة.
