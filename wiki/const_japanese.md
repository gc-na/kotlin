# Kotlinの「const」: 定数宣言の概要と使い方

## 概要
Kotlinにおける「const」は、コンパイル時に定数を定義するためのキーワードです。このキーワードを使用することで、変更不可能な値を宣言し、プログラムのパフォーマンスを向上させることができます。

## ドキュメンテーション
### 目的
「const」は、クラスのプロパティやトップレベルのプロパティとして使用できる定数を宣言するために使用されます。これにより、プログラムの状態を変更することなく、一定の値を保持し続けることが可能になります。

### 使用法
- `const`は、`val`と組み合わせて使用します。
- 定数は、基本データ型または文字列にのみ適用できます。
- `const`は、クラスの外部またはオブジェクトの内側でのみ使用可能です。

```kotlin
const val MAX_VALUE = 100
```

### 詳細
- `const`で宣言されたプロパティは、常にコンパイル時に評価されます。
- オブジェクトのインスタンス生成前に値が確定するため、グローバルな定数として利用されます。
- 変更を加えることができないため、プログラムの安全性を向上させます。

## 例
以下は、`const`の基本的な使用例です。

```kotlin
const val PI = 3.14
fun calculateCircumference(radius: Double): Double {
    return 2 * PI * radius
}
```

この例では、`PI`を定義し、円の周囲を計算する関数で使用しています。

## 説明
### 一般的な落とし穴
- `const`を使用する際は、値がコンパイル時に確定することを理解しておく必要があります。動的に変わる値には使用できません。
- `const`は、プライベートプロパティとして使用できないため、クラス内部での使用は注意が必要です。

### 注意点
- `const`は、必ず`val`とともに使用する必要があります。
- 文字列以外の複雑なオブジェクトを定数として宣言することはできません。

## 一文要約
Kotlinの「const」は、変更不可能な定数を宣言するためのコンパイル時キーワードであり、高速なプログラム実行を実現します。
