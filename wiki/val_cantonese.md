# Kotlin 中的 "val" 關鍵字：不可變變數的定義與使用

## 摘要
在 Kotlin 中，"val" 是用來聲明不可變變數的關鍵字。這意味著一旦變數被初始化後，其值就不能被改變，這有助於增強代碼的可讀性與安全性。

## 文檔
"val" 是 Kotlin 語言中用來定義只讀變數的關鍵字。與 Java 的 final 關鍵字類似，使用 "val" 聲明的變數在賦值後不能再被修改。這使得開發者可以在代碼中清楚地表達出某些值的穩定性，並減少意外更改的風險。

### 用法
使用 "val" 時，變數必須在聲明時進行初始化，否則會引發編譯錯誤。以下是基本的語法：

```kotlin
val variableName: Type = value
```

例如：
```kotlin
val pi: Double = 3.14
```

在這個例子中，變數 `pi` 被聲明為 `Double` 類型並初始化為 3.14，之後再也不能給 `pi` 賦其他值。

## 示例
以下是一些使用 "val" 的基本範例：

1. 聲明一個整數變數：
   ```kotlin
   val age: Int = 25
   ```

2. 聲明一個字符串變數：
   ```kotlin
   val name: String = "Alice"
   ```

3. 聲明一個列表：
   ```kotlin
   val numbers: List<Int> = listOf(1, 2, 3, 4, 5)
   ```

## 解釋
使用 "val" 的主要優勢是它強制要求變數在聲明時必須被初始化，從而減少了空指針異常的風險。然而，開發者在使用時也應注意以下幾點：

- **不可變性**：雖然 "val" 變數本身不可變，但如果其類型是可變的（如列表或可變集合），則可以改變該集合的內容。這並不違反 "val" 的不可變性，因為你並沒有改變變數的引用。
  
  ```kotlin
  val mutableList = mutableListOf(1, 2, 3)
  mutableList.add(4) // 合法，因為改變的是列表的內容
  ```

- **初始化要求**：必須在聲明時初始化變數，否則會導致編譯錯誤。

## 一句總結
在 Kotlin 中，"val" 用於聲明不可變變數，增強代碼的穩定性與安全性。
