# Kotlin 생성자: 객체 지향 프로그래밍의 기초

## 개요
Kotlin에서 생성자는 클래스의 인스턴스를 생성할 때 필요한 초기화 로직을 정의하는 중요한 구성 요소입니다. 생성자는 클래스의 속성을 초기화하고, 객체의 상태를 설정하는 데 사용됩니다.

## 문서화
Kotlin에서 생성자는 기본 생성자와 부 생성자로 나뉩니다. 기본 생성자는 클래스 선언부에 위치하며, 부 생성자는 클래스 본문 내에 정의됩니다.

### 기본 생성자
기본 생성자는 클래스 이름 옆에 위치하며, 클래스의 속성을 초기화할 수 있습니다. 기본 생성자는 다음과 같은 형식으로 정의됩니다:

```kotlin
class 클래스이름(속성1: 타입1, 속성2: 타입2) {
    // 클래스 본문
}
```

### 부 생성자
부 생성자는 `constructor` 키워드를 사용하여 정의됩니다. 부 생성자는 기본 생성자 외에 추가적인 초기화 로직이 필요할 때 사용됩니다. 다음은 부 생성자의 형식입니다:

```kotlin
class 클래스이름(속성1: 타입1) {
    constructor(속성2: 타입2) : this(기본값1) {
        // 초기화 로직
    }
}
```

부 생성자는 여러 개 정의할 수 있으며, 각 부 생성자는 기본 생성자를 호출해야 합니다.

## 예제
다음은 Kotlin에서 기본 생성자와 부 생성자를 사용하는 간단한 예제입니다.

```kotlin
class 사람(val 이름: String, var 나이: Int) {
    init {
        println("$이름의 나이는 $나이입니다.")
    }

    constructor(이름: String) : this(이름, 0) {
        println("나이를 지정하지 않았습니다. 기본값으로 0이 사용됩니다.")
    }
}

fun main() {
    val 사람1 = 사람("홍길동", 30)
    val 사람2 = 사람("김철수")
}
```
위 예제에서 `사람` 클래스는 기본 생성자를 통해 이름과 나이를 초기화하며, 부 생성자를 통해 이름만 입력받아 나이를 0으로 설정합니다.

## 설명
Kotlin 생성자를 사용할 때 주의해야 할 점은 다음과 같습니다:

1. **초기화 순서**: 기본 생성자의 초기화 블록은 부 생성자보다 먼저 실행됩니다.
2. **부 생성자의 호출**: 부 생성자는 반드시 기본 생성자를 호출해야 하며, 그렇지 않으면 컴파일 오류가 발생합니다.
3. **가시성**: 생성자의 가시성은 클래스의 가시성과 관련이 있으며, `public`, `private`, `protected` 등으로 설정할 수 있습니다.

또한, 생성자는 클래스의 필드 및 속성을 초기화하는 데 중요한 역할을 하므로, 명확하게 정의하고 사용해야 합니다.

## 한 줄 요약
Kotlin에서 생성자는 클래스의 객체를 초기화하는 데 필수적인 요소로, 기본 생성자와 부 생성자를 통해 다양한 초기화 방법을 제공합니다.
